## 6. Technical SEO

### Core Web Vitals
- **LCP (Largest Contentful Paint):** &lt; 2.5s. Optimize hero images (WebP/AVIF, responsive `srcset`), defer non-critical JS, use a CDN for static assets.
- **INP / FID (Interactivity):** Minimize main-thread work: code-split, lazy-load below-fold, avoid large synchronous scripts.
- **CLS (Cumulative Layout Shift):** Reserve space for images/videos (width/height or aspect-ratio), avoid injecting content above existing content; use `font-display: optional` or `swap` with reserved space for text.
- **Measure:** Use PageSpeed Insights, Search Console (Core Web Vitals report), and real-user monitoring (e.g. Vercel Analytics, Cloudflare Web Analytics).

### Page Speed
- **Hosting:** Use a fast, global CDN (e.g. Vercel, Cloudflare, AWS CloudFront).
- **Frontend:** Next.js/React: use dynamic imports, image optimization (`next/image` or equivalent), minimal critical CSS, tree-shaking.
- **Caching:** Strong cache headers for static assets; consider ISR or static generation where possible.
- **Third parties:** Load analytics and chat asynchronously; defer or lazy-load non-essential widgets.
- **Target:** &lt; 3s full load on 3G-like conditions for key landing and service pages.

### Structured Data (Schema)
- **Organization** on homepage: `Organization` + optional `LocalBusiness`.
- **Service pages:** `Service` schema (name, description, provider, areaServed).
- **Blog posts:** `Article` or `BlogPosting` (headline, datePublished, dateModified, author, image).
- **FAQ pages/sections:** `FAQPage` with `Question`/`Answer`.
- **Breadcrumbs:** `BreadcrumbList` on all relevant pages.
- **Optional:** `SoftwareApplication` for product/portfolio items if applicable.
- **Validate:** Google Rich Results Test and Search Console.

### Sitemap
- **URLs:** All service pages, pillar pages, blog posts, main landing pages; exclude params, duplicates, and noindex pages.
- **Format:** `sitemap.xml` (and `sitemap-index.xml` if multiple sitemaps).
- **Priority/freq:** Homepage + services: `priority` 0.9–1.0, `changefreq` weekly; blog: 0.7–0.8, weekly/daily; pillars: 0.8–0.9, weekly.
- **Submit:** Google Search Console and Bing Webmaster Tools; reference in `robots.txt`.

### Landing Page SEO
- **One primary keyword per page** in title, H1, and first paragraph.
- **Unique title and meta description** per page (55–60 chars title, 150–160 chars description), with CTA or benefit.
- **Clear H1 → H2 → H3** hierarchy; include keywords naturally in headings.
- **Internal links** to 2–4 relevant service/pillar pages and 1–2 high-value blog posts.
- **URLs:** Short, readable, keyword-inclusive slugs (e.g. `/services/flutter-app-development`).
- **Canonical:** One canonical URL per logical page; use `rel="canonical"` on paginated or duplicate variants.

## 7. Conversion SEO

### Landing Page Sections (Service / Lead Gen)
1. **Hero:** Headline (with primary keyword) + subhead + single strong CTA (e.g. “Get a Free Consultation”).
2. **Problem / Agitation:** Short “we understand” block (e.g. need for speed, scalability, right stack).
3. **Solution / Offer:** What you deliver (e.g. “React & Next.js web apps that scale”) + proof (logos, one-liner metrics).
4. **Process:** 3–5 steps (e.g. Discovery → Build → Launch → Iterate) to reduce uncertainty.
5. **Social proof:** Logos, short quotes, case study snippets; link to full case studies if available.
6. **Stack / Capabilities:** Icons or list of technologies (React, Flutter, Node, AWS, etc.).
7. **FAQ:** 4–6 questions targeting “cost,” “timeline,” “how it works,” “who we work with.”
8. **Final CTA:** Repeat primary CTA + secondary (e.g. “Download the MVP Checklist” or “Read our guide”).

### CTA Placement
- **Above the fold:** One primary CTA (button or link).
- **Mid-page:** After “solution” or “process” (e.g. “Discuss your project”).
- **After social proof:** “Start your project” or “Get a quote.”
- **Sticky/footer:** Optional sticky bar or footer CTA on long pages.
- **Blog:** End-of-post CTA to a service page or lead magnet; inline CTAs in relevant sections (e.g. after “MVP cost” → “Get an MVP quote”).

### Lead Capture
- **Low friction:** “Get a free consultation,” “Book a call,” “Download the MVP checklist” (email only).
- **Medium:** “Get a custom quote” or “Project planner” (short form: project type, timeline, budget range, email).
- **High value:** “Tech stack guide,” “RFP template,” “Scaling playbook” (email + optional company/role).
- **Retargeting:** Use UTM parameters and a clear thank-you page; tag visitors for paid retargeting (e.g. LinkedIn, Google).
- **Chat:** Optional chat widget with “Discuss my project” or “Get a quote” as first option; route to calendar or form.
